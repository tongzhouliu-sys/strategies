SECTION 03
Pre-screen Gate · 5项市场准入检查（全部通过才扫描）
⚠ 任意一项不通过 → 本轮扫描立即终止，不扫描任何标的
Gate-1 · 市场状态（S&P500）
❌ 不通过：SPY在MA200下方 AND 20日回撤 >10%
✅ 通过：MA200上方 OR 回撤≤10%（弱势市场降级保留）
⚠ 数据获取失败时跳过（默认通过）
Gate-2 · VIX绝对值上限
❌ 不通过：VIX > 35（恐慌区间，禁止卖方策略）
✅ 通过：VIX ≤ 35
⚠ VIX数据获取失败时跳过（默认通过）
Gate-3 · VIX期限结构（影响Delta范围）
本项不会阻断扫描，但会调整Delta区间：
正向（NORMAL）：VIX < VIX3M → Delta范围不变
⚠ 倒挂（INVERTED）：VIX > VIX3M →
  DELTA_MIN 从 -0.30 收紧至 -0.28
  （排除最激进端合约，保留安全端 -0.28 ~ -0.15）
Gate-4 · 保证金使用率
❌ 不通过：保证金使用率 ≥ 55%，禁止新开仓
✅ 通过：使用率 < 55%
⚠ 未通过 --margin-used 传入时跳过（默认通过）
Gate-5 · 三因子综合得分（需≥50分）
❌ 不通过：综合得分 < 50
✅ 通过：综合得分 ≥ 50
因子条件得分
VIX水平VIX数据不可用25
VIX < 20（低波动）35
20 ≤ VIX < 28（中波动）25
28 ≤ VIX < 35（高波动）15
S&P500位置在MA200上方30
MA200下方但20日回撤<5%15
MA200下方且回撤≥5%5
期限结构NORMAL（VIX < VIX3M）25
数据不全15
INVERTED（VIX > VIX3M）5
总分上限（理想市场） 90
SECTION 04
单标的处理管道（process_ticker）
每个标的在独立线程中串行执行以下6个步骤，任一步骤失败则该标的标记 ERROR 并跳过。
① 股价 + 技术均线
获取1年历史价格 → 计算 MA20 / MA50 / MA200、52周高低、5日/3日跌幅、连续收阴天数
调用 get_stock_data()，失败直接返回 ERROR
② IVR计算（隐含波动率百分位）
优先：从 SQLite iv_history.db 读取真实IV历史，计算 IVR = (当前IV - 年低) / (年高 - 年低) × 100
回退：HV代理 — 用历史价格计算20日滚动HV代替IV，标记 hv_proxy ⚠
同时计算 iv_trend：近5条均值 vs 前5条均值，差异>5% → rising/falling，否则 flat
若 IVR=None（数据不足30条）→ ivr_meets=None → 最终不产生信号
③ 财报 / FOMC 黑名单检查
in_blackout：距财报日 ≤ earnings_blackout天 → 禁止开仓
保守处理：blackout>0 且财报日获取失败 → 强制 in_blackout=True，标记 earnings_unknown_risk
in_fomc_blackout：距下一次FOMC ≤ fomc_blackout天 → 禁止（仅SPY/QQQ/XLU生效）
near_blackout（财报前7天缓冲警告）、post_earnings_vol（财报后5天IV崩塌窗口）
④ Opportunity Alert 评估
评估5项条件（详见下一节），输出 triggered / partial 信号
触发后仍须完整走完后续步骤，并在开仓时 仓位 × 50%，须 尾盘30分钟确认
⑤ 期权链获取 + 到期日选择
select_expiry()：优先月度（第3周五）DTE 35-45天，其次周到期，最后保底30-60天，最接近40天
get_put_chain()：拉取Put期权链，执行质量过滤（见合约筛选节）
同步将最接近ATM的IV存入 iv_history.db
⑥ 最优合约选择 + 信号判定
find_best_contracts()：满足Delta区间 + 年化收益 + OTM深度条件，排序后取Top 3
has_signal：综合判断是否产生信号（见信号判定节）
SECTION 05
Opportunity Alert · 5项联合触发条件
用于识别"标的短期急跌 + IV同步升温"的买入窗口。5项全部满足 → triggered=True（推荐50%仓位尾盘开仓）
仅C1满足其余未全满足 → partial=True（仅供人工参考，不产生推荐信号）
C1
跌幅触发（主条件）
5日跌幅 ≥ 阈值 OR 3日跌幅 ≥ 阈值
满足任一即C1=True
（v2.0新增，替代原连跌天数）
C2
IVR 5日涨幅
IVR较5日前上升 ≥ 15个百分点
从 SQLite 读取5交易日前的IVR
若历史数据不足则C2=False
C3
无黑名单约束
not in_blackout AND not in_fomc_blackout
财报禁区或FOMC禁区内 → C3=False → 不触发Alert
C4
标的评级门槛
grade ∈ {A+, A, B}
C级高波标的不触发Opportunity Alert
（C级阈值更高，本身已差异化）
C5
SPX未创20日新低
S&P500今收 < 过去20交易日最低价 → C5=False
判断是个股下跌而非系统性崩盘
窗口不含当天（v2.1 bug修复）
评级 C1 · 5日跌幅门槛 C1 · 3日跌幅门槛 推荐结构 触发后仓位
A+ ≥ 5.0% ≥ 3.0% CSP ×50%（尾盘确认）
A ≥ 5.0% ≥ 3.0% CSP ×50%（尾盘确认）
B ≥ 6.0% ≥ 4.0% CSP ×50%（尾盘确认）
C（参考） ≥ 12.0% ≥ 8.0% Bull Put Spread C4=False，不触发
SECTION 06
合约筛选与排序（get_put_chain + find_best_contracts）
第一阶段：质量过滤（硬性门槛，任一不满足直接丢弃）
行权价范围
股价 × (1 - OTM_buffer×0.5) 以下
股价 × (1 - OTM_buffer×2.0) 以上
（OTM缓冲带±范围）
→
有效报价
bid > 0 AND ask > 0
（零价合约强制跳过）
IV > 0
→
价差过滤
(ask-bid)/mid × 100
≤ MAX_SPREAD_PCT（默认15%）
→
持仓量 OI
OI ≥ MIN_OI（默认50）
流动性保证
第二阶段：Greeks计算
📡 真实 Greeks（优先）
API返回真实 delta/theta
直接使用，精度最高
theta_seller = -theta_raw × 100
OR
📐 Black-Scholes 估算（兜底）
iv_raw归一化：≥1.0时÷100
（v8.0 Bug修复：原来直接用30.0传入BS，等于3000%波动率）
bs_put_delta(S, K, T, r, σ)
第三阶段：合格性筛选（find_best_contracts）
Delta 区间
Delta_min ≤ delta ≤ Delta_max
默认：-0.30 ~ -0.15
期限结构倒挂时 min→-0.28
AND
年化收益
(mid/strike) × (365/DTE) × 100
≥ ann_min（标的配置）
AND
OTM深度
otm_pct ≥ otm_buffer × 100
A+/A：≥8% / B：≥5% / C：≥5%
第四阶段：排序规则（v8.0重构，安全优先）
1
OTM深度（主要）
otm_pct 越大越靠前
先保安全垫，再优化收益
2
年化收益（次要）
相同OTM深度时，
年化收益高者优先
3
OI流动性（第三）
持仓量越高
流动性越好
✅ 最终输出：Top 3合约（LLM精简模式只输出Top 1）
SECTION 07
has_signal · 最终信号判定逻辑
以下所有条件必须同时为 True，才产生 has_signal=True 信号
🚫
not in_blackout
未在财报禁区内（知道日期时按天数判断；财报日未知时保守处理为在禁区）
📅
not in_fomc_blackout
未在FOMC会议前禁入窗口内（仅对配置了fomc_blackout的标的生效）
🥇
not gld_vix_blocked
GLD特殊规则：VIX < 22 时黄金期权禁止开仓（VIX太低时黄金无保险价值）
📉
not (block_post_earnings AND post_earnings_vol)
仅在开启 --block-post-earnings 时生效：财报后5天IV崩塌窗口内阻断信号
📊
ivr_meets is True（明确为True，不接受None）
IVR ≥ 标的最低IVR要求（不同标的阈值不同）；IVR=None（数据不足）不等于通过
✅
len(best_contracts) > 0
至少存在1个满足Delta+年化+OTM条件的合格合约
📋 警告类标记（不阻断信号，仅在输出中标注）：
· ivr_hv_proxy：IVR来自HV代理而非真实IV历史（默认允许，--block-hv-proxy 可硬阻断）
· iv_trend_rising：IV处于上升趋势（默认允许，--block-rising-iv 可硬阻断）
· near_blackout：距财报禁区<7天，提示人工注意
· earnings_unknown_risk：财报日获取失败，保守处理（实际已作为阻断条件）
